# Contributing to Sanehood Feedback

Thanks for helping improve Sanehood.

This repository is the public place for user-facing feedback, bug reports, feature requests, and usability observations. It is not the main development repository, and public code contributions are not the goal here.

## Before you open an issue

Please:

1. Search existing issues first to avoid duplicates.
2. Use the most relevant issue template.
3. Open one issue per bug, request, or concern.
4. Include enough detail for someone else to understand the problem.
5. Avoid posting private, sensitive, or account-specific information.

## What belongs here

Good fits include:

- bugs or broken behavior
- confusing or frustrating user experiences
- accessibility issues
- performance concerns
- copy or content improvements
- feature ideas from a user perspective

Not a fit:

- internal implementation proposals
- support requests that require private account details
- security disclosures shared publicly
- code pull requests for this repository

## How to write a helpful issue

A strong issue usually includes:

- a short summary
- steps to reproduce, if relevant
- expected behavior
- what actually happened
- browser, device, or app version information
- screenshots or recordings, if helpful

The issue forms in `.github/ISSUE_TEMPLATE/` are designed to collect this information.

## Labels and triage

Maintainers will use public labels to track status and type. You may see labels such as:

- `status/needs-triage`
- `status/needs-info`
- `status/confirmed`
- `status/planned`
- `status/in-progress`
- `status/awaiting-feedback`
- `status/shipped`
- `status/declined`
- `status/duplicate`
- `type/bug`
- `type/feature`
- `type/improvement`
- `type/feedback`

## Pull requests

This repository is not intended for public code changes. If you have feedback, please open an issue instead.

## Security issues

Do not report security issues here. Use the private security reporting process described in `SECURITY.md`.
