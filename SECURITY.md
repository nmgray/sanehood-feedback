# Security Policy

## Reporting a Security Issue

Please do not open public issues for security vulnerabilities or sensitive account-related concerns.

Use the private security disclosure process for Sanehood instead:

support@sanehood.com

## What to include

When reporting a security issue privately, please include as much detail as possible, such as:

- a clear description of the issue
- affected feature, page, or flow
- steps to reproduce or demonstrate the concern
- the potential impact
- screenshots, logs, or examples if they help explain the problem

## What to avoid

Do not include sensitive personal data, secrets, tokens, or account credentials in a public issue.

## Expected response

Security reports are reviewed privately and handled separately from public feedback issues.
